# Firebase Data Structure Example

This document shows example data structures for the content collections in Firebase Firestore.

## Collection Structure

```
Firestore Root
└── content (collection)
    ├── locations (document)
    │   ├── chai (subcollection)
    │   ├── changnoi (subcollection)
    │   ├── mali (subcollection)
    │   ├── ping (subcollection)
    │   └── plakad (subcollection)
    ├── foodMatches (document)
    │   ├── chai (subcollection)
    │   ├── changnoi (subcollection)
    │   ├── mali (subcollection)
    │   ├── ping (subcollection)
    │   └── plakad (subcollection)
    └── festivalFits (document)
        ├── chai (subcollection)
        ├── changnoi (subcollection)
        ├── mali (subcollection)
        ├── ping (subcollection)
        └── plakad (subcollection)
```

## Example Document Structures

### Location Document (in `content/locations/ping/`)

```json
{
  "name": "Railay Beach",
  "province": "Krabi",
  "region": "Southern Thailand",
  "description": "A stunning peninsula accessible only by boat, famous for rock climbing, clear waters, and dramatic limestone cliffs. Perfect for kayaking, snorkeling, and beach adventures.",
  "imageUrl": "https://firebasestorage.googleapis.com/v0/b/tat-character-questionnaire.firebaseusercontent.com/o/characters%2FUnitedKingdom%2Fping%2Frailay.jpg?alt=media&token=...",
  "googleMapLink": "https://maps.app.goo.gl/xyz123",
  "createdAt": "2026-01-13T12:00:00Z",
  "updatedAt": "2026-01-13T12:00:00Z"
}
```

**Required Fields:**
- `name` (string) - Location name
- `province` (string) - Province or city name
- `region` (string) - Region of Thailand
- `description` (string) - Detailed description

**Optional Fields:**
- `imageUrl` (string) - Firebase Storage URL or external image URL
- `googleMapLink` (string) - Direct Google Maps link
- `createdAt` (timestamp) - Document creation time
- `updatedAt` (timestamp) - Last update time

### Food Document (in `content/foodMatches/ping/`)

```json
{
  "name": "Pla Nueng Manao (Steamed Fish with Lime)",
  "description": "Fresh fish steamed with lime, garlic, and chili, creating a zesty and aromatic dish. It's light yet packed with bold flavors — perfect for those who love seafood and citrusy kicks.",
  "imageUrl": "https://firebasestorage.googleapis.com/v0/b/tat-character-questionnaire.firebaseusercontent.com/o/food%2Fping%2Fpla-nueng-manao.jpg?alt=media&token=...",
  "createdAt": "2026-01-13T12:00:00Z",
  "updatedAt": "2026-01-13T12:00:00Z"
}
```

**Required Fields:**
- `name` (string) - Dish name (can include Thai name in parentheses)
- `description` (string) - Food description

**Optional Fields:**
- `imageUrl` (string) - Firebase Storage URL or external image URL
- `thaiName` (string) - Thai script name if separate
- `createdAt` (timestamp) - Document creation time
- `updatedAt` (timestamp) - Last update time

### Festival Document (in `content/festivalFits/ping/`)

```json
{
  "name": "Full Moon Party (Koh Phangan)",
  "description": "An iconic beach party held every full moon on Haad Rin Beach. It features fire shows, live DJs, and non-stop dancing under the stars. It's a must for thrill-seekers and party lovers.",
  "festivalPeriod": "Monthly (Full Moon)",
  "imageUrl": "https://firebasestorage.googleapis.com/v0/b/tat-character-questionnaire.firebaseusercontent.com/o/festivals%2Fping%2Ffull-moon-party.jpg?alt=media&token=...",
  "createdAt": "2026-01-13T12:00:00Z",
  "updatedAt": "2026-01-13T12:00:00Z"
}
```

**Required Fields:**
- `name` (string) - Festival/event name
- `description` (string) - Event description

**Optional Fields:**
- `festivalPeriod` (string) - When it occurs (e.g., "April", "November", "Year-round")
- `imageUrl` (string) - Firebase Storage URL or external image URL
- `location` (string) - Where it takes place
- `createdAt` (timestamp) - Document creation time
- `updatedAt` (timestamp) - Last update time

## Image URL Format

Images should be stored in Firebase Storage and the URL should follow this pattern:

```
https://firebasestorage.googleapis.com/v0/b/[PROJECT_ID].appspot.com/o/[PATH]?alt=media&token=[TOKEN]
```

Example paths:
- Locations: `locations/ping/railay-beach.jpg`
- Food: `food/ping/pla-nueng-manao.jpg`
- Festivals: `festivals/ping/full-moon-party.jpg`

## Document ID

Document IDs can be auto-generated by Firebase or custom strings. The app doesn't rely on specific document IDs, so use whatever makes sense for your data management:

- Auto-generated: `7rOJkMXD6iI5qoEhFxD4`
- Custom: `railay-beach`, `pla-nueng-manao`, etc.

## Character-Specific Data

Each character should have their own set of documents in their respective subcollections:

### Ping (Adventure Seeker)
- **Locations**: Beach destinations, diving spots, adventure parks
- **Food**: Seafood, spicy dishes, beach eats
- **Festivals**: Beach parties, water festivals, adventure events

### Mali (Aesthetic Explorer)
- **Locations**: Cafés, art galleries, design districts, rooftop bars
- **Food**: Aesthetic dishes, trendy café food, photogenic desserts
- **Festivals**: Art festivals, design events, contemporary celebrations

### Chai (Mindful Wanderer)
- **Locations**: Nature spots, quiet temples, riverside cafés
- **Food**: Comfort food, traditional dishes, simple meals
- **Festivals**: Spiritual events, meditation retreats, peaceful celebrations

### Chang-Noi (Culture Keeper)
- **Locations**: Historical temples, traditional villages, local markets
- **Food**: Traditional dishes, regional specialties, heritage recipes
- **Festivals**: Traditional festivals, cultural ceremonies, heritage events

### Pla-Kad (Luxury Connoisseur)
- **Locations**: Luxury hotels, fine dining restaurants, high-end spas
- **Food**: Gourmet dishes, fine dining, premium ingredients
- **Festivals**: Exclusive events, film festivals, jazz concerts

## Migration from Hardcoded Data

If you're migrating from the hardcoded data in the app, here's a mapping:

1. Old `_getCharacterSpots()` → New `content/locations/{character}/`
2. Old `_getCharacterFoods()` → New `content/foodMatches/{character}/`
3. Old `_getCharacterEvents()` → New `content/festivalFits/{character}/`

The app will automatically fetch from Firebase and display the data with proper formatting, images, and links.
